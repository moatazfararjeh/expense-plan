# النسخ الاحتياطي لقاعدة البيانات
# Database Backup Guide

## 📁 ملفات النسخ الاحتياطي / Backup Scripts

تم إنشاء ٣ سكريبتات لإدارة النسخ الاحتياطي:

| الملف | الوظيفة | الاستخدام |
|------|---------|----------|
| `backup-database.ps1` | إنشاء نسخة احتياطية | `.\backup-database.ps1` |
| `restore-database.ps1` | استعادة نسخة احتياطية | `.\restore-database.ps1` |
| `schedule-backup.ps1` | جدولة النسخ التلقائي | `.\schedule-backup.ps1` (يتطلب صلاحيات Admin) |

---

## 🔄 إنشاء نسخة احتياطية يدوياً
### Manual Backup

لإنشاء نسخة احتياطية فورية:

```powershell
.\backup-database.ps1
```

**ماذا يفعل؟**
- ينشئ ملف SQL من قاعدة البيانات الحالية
- يحفظ الملف في: `C:\Mutaz\Expense Plan\backups\`
- اسم الملف يحتوي على التاريخ: `expense_plan_20240115_143022.sql`
- يحتفظ بآخر ١٠ نسخ فقط (يحذف الأقدم تلقائياً)

**متطلبات:**
- PostgreSQL يجب أن يكون في PATH
- كلمة المرور: `P@ssw0rd`

---

## ♻️ استعادة نسخة احتياطية
### Restore Backup

لاستعادة قاعدة البيانات من نسخة سابقة:

```powershell
.\restore-database.ps1
```

**خطوات الاستعادة:**
1. السكريبت يعرض قائمة بجميع النسخ المتوفرة
2. اختر رقم النسخة التي تريد استعادتها
3. اكتب `YES` للتأكيد (يحذف البيانات الحالية!)
4. ينتظر حتى تكتمل الاستعادة

**⚠️ تحذير:**
- هذه العملية **تحذف** قاعدة البيانات الحالية بالكامل
- تأكد من إنشاء نسخة احتياطية حديثة قبل الاستعادة

---

## ⏰ جدولة النسخ الاحتياطي التلقائي
### Schedule Automatic Backup

لتفعيل النسخ الاحتياطي التلقائي:

```powershell
# افتح PowerShell كـ Administrator
.\schedule-backup.ps1
```

**خيارات الجدولة:**
1. **يومياً في منتصف الليل** - نسخة كل يوم الساعة ١٢ صباحاً
2. **يومياً في وقت محدد** - اختر الوقت الذي تريده
3. **أسبوعياً** - نسخة كل يوم أحد في منتصف الليل
4. **إلغاء الجدولة** - إيقاف النسخ التلقائي

**بعد الإعداد:**
- النسخ الاحتياطي يعمل تلقائياً في الخلفية
- لا يحتاج تشغيل يدوي
- يمكن مراجعة الجدولة من Task Scheduler

**لعرض/تعديل الجدولة:**
1. افتح Task Scheduler: اكتب `taskschd.msc` في Run
2. افتح Task Scheduler Library
3. ابحث عن: `ExpensePlan-DatabaseBackup`

## 🔧 استكشاف الأخطاء
### Troubleshooting

### خطأ: pg_dump not found

**السبب:** PostgreSQL غير موجود في PATH

**الحل السريع:**
```powershell
# استخدم السكريبت التلقائي
.\add-postgres-path.ps1
```

**أو يدوياً:**
```powershell
# أضف PostgreSQL للـ PATH:
$env:Path += ";C:\Program Files\PostgreSQL\18\bin"
```

أو أضفه بشكل دائم:
1. ابحث عن "Environment Variables" في Windows
2. عدّل على PATH
3. أضف: `C:\Program Files\PostgreSQL\[version]\bin`

---

### خطأ: Authentication failed

**السبب:** كلمة المرور خاطئة

**الحل:**
عدّل كلمة المرور في السكريبتات:
```powershell
$env:PGPASSWORD = "كلمة_المرور_الصحيحة"
```

---

### خطأ: Cannot drop database (في الاستعادة)

**السبب:** قاعدة البيانات قيد الاستخدام

**الحل:**
1. أوقف تشغيل IIS sites:
```powershell
Stop-IISSite ExpensePlanAPI
Stop-IISSite ExpensePlanApp
```

2. نفذ الاستعادة

3. أعد تشغيل IIS sites:
```powershell
Start-IISSite ExpensePlanAPI
Start-IISSite ExpensePlanApp
```

## ❓ أسئلة شائعة
### FAQ

**س: هل يؤثر النسخ على أداء الموقع؟**
ج: لا، النسخ يأخذ ثوانٍ معدودة ولا يوقف الموقع.

**س: ماذا لو امتلأ القرص من النسخ؟**
ج: السكريبت يحتفظ بآخر ١٠ نسخ فقط، ويحذف القديمة.

**س: هل يمكن استعادة نسخة قديمة جداً؟**
ج: نعم، إذا كانت محفوظة في `backups\` ستظهر في القائمة.

**س: ماذا لو نسيت كلمة المرور؟**
ج: ستحتاج إعادة تعيين كلمة مرور PostgreSQL من pgAdmin.
