# حل مشكلة التشفير ✅

## المشكلة التي واجهتها:

عند محاولة تشفير البيانات، ظهرت الرسالة التالية:
```
error: invalid input syntax for type numeric: "xxxxx:yyyyy:zzzzz"
```

## سبب المشكلة:

أعمدة المبالغ في قاعدة البيانات PostgreSQL كانت من نوع `NUMERIC` وهذا النوع يقبل الأرقام فقط. لكن القيم المشفرة هي نصوص (strings) بصيغة `iv:encrypted:tag`.

## الحل:

تم إنشاء سكريبت جديد لتحويل نوع الأعمدة من `NUMERIC` إلى `TEXT` قبل التشفير.

---

## 🚀 كيفية تفعيل التشفير الآن:

### الطريقة السهلة (موصى بها):

استخدم السكريبت الشامل الجديد:

```powershell
.\setup-encryption.ps1
```

هذا السكريبت سيقوم بـ:
1. ✓ تحديث بنية قاعدة البيانات تلقائياً
2. ✓ تشفير جميع البيانات الموجودة
3. ✓ عرض تقرير مفصل عن نتائج العملية

### الطريقة اليدوية:

إذا أردت التحكم الكامل:

```powershell
# الخطوة 1: تحديث قاعدة البيانات
cd backend
node updateDatabaseForEncryption.js

# الخطوة 2: تشفير البيانات
node encryptExistingData.js
```

## 📝 ملاحظات مهمة:

1. **أوقف خادم backend قبل التشغيل**
   ```powershell
   # اضغط Ctrl+C في نافذة الخادم
   ```

2. **تأكد من مفتاح التشفير**
   - افتح `backend\.env`
   - تحقق من وجود `ENCRYPTION_KEY`
   - غيّره إلى قيمة سرية خاصة بك

3. **النسخ الاحتياطي**
   - من المستحسن عمل backup لقاعدة البيانات قبل التشغيل
   - استخدم `.\backup-database.ps1` إذا كان متوفراً

---

## ✅ بعد التشفير:

1. أعد تشغيل خادم backend:
   ```powershell
   .\start-backend.ps1
   ```

2. افتح التطبيق وتحقق من أن كل شيء يعمل بشكل طبيعي

3. افتح قاعدة البيانات وتحقق من أن المبالغ مشفرة:
   ```sql
   SELECT amount FROM monthly_expenses LIMIT 1;
   -- يجب أن ترى شيئاً مثل: "a1b2c3:d4e5f6:g7h8i9"
   ```
