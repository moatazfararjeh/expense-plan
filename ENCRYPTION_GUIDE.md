# نظام تشفير المبالغ المالية

## نظرة عامة

تم إضافة نظام تشفير آمن لحماية جميع المبالغ المالية في قاعدة البيانات باستخدام خوارزمية AES-256-GCM.

## المبالغ المشفرة

يتم تشفير الحقول التالية:

### 1. user_settings
- `monthly_salary` - الراتب الشهري
- `opening_balance` - الرصيد الافتتاحي

### 2. monthly_expenses
- `amount` - مبلغ المصروف الشهري

### 3. daily_transactions
- `amount` - مبلغ المعاملة اليومية

### 4. additional_income
- `amount` - مبلغ الدخل الإضافي

### 5. salary_changes
- `amount` - مبلغ تغيير الراتب

## خطوات التفعيل

### 1. تحديث مفتاح التشفير (مهم جداً!)

في ملف `.env`، قم بتغيير `ENCRYPTION_KEY` إلى قيمة سرية فريدة ومعقدة:

```env
ENCRYPTION_KEY=your-very-secure-and-long-encryption-key-here
```

**تحذير:** 
- يجب تغيير المفتاح الافتراضي قبل الاستخدام في بيئة الإنتاج
- لا تشارك هذا المفتاح مع أي شخص
- احتفظ بنسخة احتياطية آمنة من المفتاح
- إذا فقدت المفتاح، لن تتمكن من فك تشفير البيانات المخزنة

### 2. تحديث قاعدة البيانات وتشفير البيانات

استخدم السكريبت الشامل الذي يقوم بكل شيء تلقائياً:

```powershell
.\setup-encryption.ps1
```

هذا السكريبت سوف:
1. **تحديث بنية قاعدة البيانات**: تحويل أعمدة المبالغ من `NUMERIC` إلى `TEXT`
2. **تشفير البيانات الموجودة**: قراءة وتشفير جميع المبالغ المالية
3. **التحقق من النجاح**: عرض تقرير شامل عن العملية

**ملاحظة:** 
- تأكد من إيقاف خادم backend قبل تشغيل السكريبت
- السكريبت يتحقق من البيانات المشفرة مسبقاً ويتجنب تشفيرها مرة أخرى
- يمكن تشغيله بأمان عدة مرات

### البديل: تشغيل الخطوات يدوياً

إذا أردت التحكم الكامل، يمكنك تشغيل الخطوات يدوياً:

```powershell
cd backend

# الخطوة 1: تحديث بنية قاعدة البيانات
node updateDatabaseForEncryption.js

# الخطوة 2: تشفير البيانات
node encryptExistingData.js
```

### 3. إعادة تشغيل الخادم

بعد تشفير البيانات، أعد تشغيل خادم Backend:

```powershell
cd backend
npm start
```

## كيف يعمل

### التشفير
عند حفظ أي مبلغ مالي:
1. يتم تحويل الرقم إلى نص
2. يتم توليد IV (Initialization Vector) عشوائي
3. يتم تشفير النص باستخدام AES-256-GCM
4. يتم حفظ النتيجة بالصيغة: `iv:encrypted:tag`

### فك التشفير
عند قراءة أي مبلغ مالي:
1. يتم فصل أجزاء القيمة المشفرة (IV, encrypted, tag)
2. يتم فك التشفير باستخدام المفتاح
3. يتم تحويل النتيجة إلى رقم
4. يتم إرجاع الرقم إلى التطبيق

### الأمان
- **AES-256-GCM**: خوارزمية تشفير عسكرية المستوى
- **Authentication Tag**: يمنع التلاعب بالبيانات المشفرة
- **Random IV**: يضمن أن نفس الرقم يُشفر بطرق مختلفة في كل مرة
- **Backward Compatibility**: يدعم قراءة البيانات غير المشفرة للتوافق مع البيانات القديمة

## الصيانة

### النسخ الاحتياطي
- احتفظ دائماً بنسخة احتياطية من مفتاح التشفير
- النسخ الاحتياطي لقاعدة البيانات يجب أن يتضمن البيانات المشفرة
- يمكنك استعادة البيانات فقط إذا كان لديك نفس مفتاح التشفير

### تغيير مفتاح التشفير
إذا احتجت لتغيير مفتاح التشفير:
1. احتفظ بنسخة من المفتاح القديم
2. غيّر المفتاح في `.env`
3. ستحتاج لكتابة سكريبت خاص لفك التشفير بالمفتاح القديم وإعادة التشفير بالمفتاح الجديد

## الاختبار

للتحقق من عمل التشفير:

1. أضف مصروف أو معاملة جديدة عبر التطبيق
2. تحقق من قاعدة البيانات مباشرة:
   ```sql
   SELECT * FROM monthly_expenses LIMIT 1;
   ```
3. يجب أن ترى حقل `amount` يحتوي على نص مشفر بصيغة: `xxxxxx:yyyyyy:zzzzzz`
4. افتح التطبيق، يجب أن ترى الأرقام بشكل طبيعي

## استكشاف الأخطاء

### المشكلة: "invalid input syntax for type numeric"
```
error: invalid input syntax for type numeric: "xxxxx:yyyyy:zzzzz"
```
**السبب:** أعمدة المبالغ في قاعدة البيانات لا تزال من نوع NUMERIC

**الحل:** شغّل سكريبت تحديث قاعدة البيانات أولاً:
```powershell
cd backend
node updateDatabaseForEncryption.js
```
أو استخدم السكريبت الشامل:
```powershell
.\setup-encryption.ps1
```

### المشكلة: "Failed to encrypt value"
- تحقق من أن مفتاح التشفير موجود في `.env`
- تأكد من أن القيمة المراد تشفيرها صحيحة

### المشكلة: "Failed to decrypt value" أو قيم NULL
- تحقق من أن مفتاح التشفير هو نفسه المستخدم للتشفير
- تحقق من أن البيانات لم تتلف

### المشكلة: القيم تظهر كأرقام عادية في قاعدة البيانات
- قد تكون نسيت تشغيل `encryptExistingData.js`
- أو قد يكون هناك خطأ في الكود

### المشكلة: الخادم لا يعمل بعد التشفير
- تأكد من إعادة تشغيل الخادم بعد التشفير
- تحقق من سجلات الخادم (Server logs) لمعرفة الخطأ
- تأكد من أن مفتاح التشفير موجود في `.env`

## ملاحظات الأمان

⚠️ **مهم جداً:**
- لا تشارك مفتاح التشفير مع أي شخص
- لا تحفظ مفتاح التشفير في Git أو أي نظام تحكم بالإصدارات
- استخدم متغيرات البيئة لتخزين المفتاح
- غيّر المفتاح الافتراضي فوراً
- احتفظ بنسخة احتياطية آمنة من المفتاح في مكان منفصل

## الدعم

في حال واجهت أي مشاكل، تحقق من:

2. قاعدة البيانات للتأكد من وجود البيانات المشفرة
3. ملف `.env` للتأكد من وجود مفتاح التشفير
